Detailed Description

Declares an add-in assembly import

An add-in may be composed by several assemblies and data files. Assemblies must be declared in the main assembly using this attribute, or in the XML manifest.

It is important to properly declare all files used by an add-in. For example, when a type from the add-in is required (e.g. an ICommand implementation), only properly declared assemblies will be checked. This information is also used by setup tools to know exactly what needs to be packaged when creating an add-in package, or to know what needs to be deleted when removing an add-in.
